# Folder Structure

```
vesion/
├── apps/
│   ├── api/                          # NestJS REST API
│   │   ├── prisma/
│   │   │   ├── schema.prisma         # data model (source of truth)
│   │   │   ├── migrations/           # generated by `pnpm db:migrate`
│   │   │   └── seed.ts               # demo data + accounts
│   │   ├── src/
│   │   │   ├── main.ts               # bootstrap: helmet, CORS, versioning, Swagger
│   │   │   ├── app.module.ts         # module registry + global guards/interceptors
│   │   │   ├── config/               # env loading + boot-time validation
│   │   │   ├── database/             # PrismaService
│   │   │   ├── infrastructure/
│   │   │   │   ├── mail/             # BullMQ queue + nodemailer + HTML template
│   │   │   │   └── redis/            # client + cache-aside helper
│   │   │   ├── common/               # cross-cutting layer
│   │   │   │   ├── decorators/       # @Public, @Roles, @CurrentUser
│   │   │   │   ├── guards/           # JWT (global), RBAC
│   │   │   │   ├── interceptors/     # response envelope, audit trail
│   │   │   │   ├── filters/          # HTTP + Prisma exception mapping
│   │   │   │   ├── dto/ utils/ types/
│   │   │   └── modules/              # one folder per bounded context
│   │   │       ├── auth/  users/  companies/
│   │   │       ├── projects/  tasks/  files/
│   │   │       ├── invoices/  payments/  subscriptions/  quotes/  contracts/
│   │   │       ├── tickets/  knowledge-base/  messages/  notifications/
│   │   │       ├── blog/  portfolio/  careers/  contact/
│   │   │       ├── ai/               # local assistant engine + SSE streaming
│   │   │       ├── analytics/  audit/  settings/  health/
│   │   │       └── <module>/{module,controller,service,dto/}
│   │   ├── test/                     # supertest e2e + jest-e2e config
│   │   └── Dockerfile
│   │
│   └── web/                          # Next.js 14 (App Router)
│       ├── src/
│       │   ├── app/
│       │   │   ├── layout.tsx  globals.css  icon.tsx  error.tsx  not-found.tsx
│       │   │   ├── robots.ts  sitemap.ts
│       │   │   ├── (marketing)/      # public site (SSR/ISR)
│       │   │   │   ├── page.tsx      # home: 3D universe experience
│       │   │   │   ├── about/ services/ portfolio/[slug] case-studies/[slug]
│       │   │   │   ├── pricing/ blog/[slug] careers/[slug] contact/ faq/
│       │   │   │   └── privacy-policy/ terms-of-service/ cookie-policy/
│       │   │   ├── (auth)/           # login, register, verify, forgot/reset
│       │   │   ├── dashboard/        # client portal (14 pages)
│       │   │   └── admin/            # back office (16 pages)
│       │   ├── components/
│       │   │   ├── ui/               # design-system kit (buttons, tables, modals…)
│       │   │   ├── marketing/        # hero, bento, universe, pipeline, orbs, nav…
│       │   │   ├── three/            # React Three Fiber scenes
│       │   │   ├── dashboard/        # shell, page header, notifications bell
│       │   │   ├── features/         # kanban, invoice/project detail, messenger…
│       │   │   ├── chat/             # AI chat widget
│       │   │   └── markdown.tsx      # injection-safe renderer
│       │   ├── lib/                  # api client, auth store, hooks, types, utils
│       │   ├── content/              # static marketing copy (services, team…)
│       │   └── test/                 # vitest setup
│       ├── e2e/                      # Playwright specs
│       └── Dockerfile
│
├── nginx/                            # production reverse proxy + TLS
├── scripts/backup.sh                 # manual dump/restore helper
├── docs/                             # this documentation set
├── .github/workflows/ci.yml          # lint → test → e2e → build → docker
├── docker-compose.yml                # dev infra (postgres, redis, mailpit)
├── docker-compose.prod.yml           # full production stack + nightly backups
├── .env.example                      # every variable, documented
├── pnpm-workspace.yaml  turbo.json   # monorepo tooling
└── README.md
```

**Rules of thumb** — new backend feature: new folder under `modules/`; UI shared by both
dashboards: `components/features/`; pure presentation: `components/ui/`; marketing-only
sections: `components/marketing/`.
